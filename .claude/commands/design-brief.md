# design-brief

You are running the **Design Brief** workflow in LEGO Agent OS.

Goal:
Create a comprehensive creative brief for a design project that provides strategic direction, visual territory, and clear requirements. This workflow produces actionable briefs for designers.

Primary Agent:
- @design-strategist

Supporting Agents:
- @brand-strategist (for strategy context)
- @marketing-strategist (for campaign context)

Skills:
- Design Direction
- Visual Identity
- Image Prompts

Standards:
- `standards/signalos/transmission-framework.md`
- `standards/signalos/brand-strategy.md`

---

## Steps

### 1. Project Scoping

Define the project:

```
PROJECT OVERVIEW:

NAME: [Project name]
TYPE: [Logo/Campaign/Website/Collateral/etc.]
DELIVERABLES: [What's needed]
TIMELINE: [Key dates]
BUDGET: [If relevant]
```

---

### 2. Strategic Context

Ground the project in strategy:

```
STRATEGIC CONTEXT:

BRAND FOUNDATION:
Archetype: [Primary + Secondary]
Positioning: [Onliness statement]
Values: [Key values]
Tribe: [Who this is for]

PROJECT PURPOSE:
Why this exists: [Background]
What it needs to achieve: [Objectives]
How we measure success: [KPIs]

AUDIENCE:
Primary: [Who specifically]
Their needs: [What they want]
Their context: [Where/when they see this]
```

### 3. Design Direction

- [Descriptor 3]: [What it means]

### 5. Requirements

```

- [Required element 2]
- [Logo usage requirements]

CONTENT:
Copy provided: [Yes/No/TBD]

Other assets: [What else]
```

### 6. Boundaries

Define what to avoid:

```
BOUNDARIES:

DO NOT:
- [Anti-pattern 1]
- [Anti-pattern 2]
- [Anti-pattern 3]

AVOID RESEMBLANCE TO:
- [Competitor/style to differentiate from]

NEVER:
- [Absolute restrictions]
```

---

### 7. Success Criteria

Define how to evaluate:

```
SUCCESS CRITERIA:

STRATEGIC:
- [ ] Expresses brand archetype
- [ ] Communicates key messages
- [ ] Appeals to target audience
- [ ] Differentiates from competitors

EXECUTION:
- [ ] Meets technical requirements
- [ ] Works across applications
- [ ] Maintains brand consistency
- [ ] Achieves quality standard

SUBJECTIVE:
- [ ] Feels [emotional territory]
- [ ] Looks [visual territory]
```

## Quality Checklist

**Completeness**
- [ ] Project clearly defined
- [ ] Strategic context provided
- [ ] Design direction given
- [ ] References curated
- [ ] Requirements specified
- [ ] Boundaries set
- [ ] Success criteria defined

**Clarity**
- [ ] Direction is actionable
- [ ] No conflicting guidance
- [ ] Priorities are clear
- [ ] Questions are flagged

**Strategic Grounding**
- [ ] Aligned with brand
- [ ] Serves project purpose
- [ ] Appeals to audience
